# Screen Reader Accessibility Component\n\n## Overview\n\nThe Screen Reader Component provides text-to-speech functionality for users with visual impairments or those who prefer audio content. It features a persistent, hovering icon on the left side of the screen that can be activated to read page content aloud.\n\n## Features\n\n- **Floating Icon Interface**: Fixed position icon on the left side of the screen at 50% vertical position\n- **Text-to-Speech**: Reads main page content using Web Speech API\n- **Playback Controls**: Play, Pause, Resume, and Stop buttons\n- **Configurable Settings**:\n  - Speech rate (0.5 to 2.0)\n  - Language selection (default: en-US)\n  - Automatic content extraction\n- **Browser Support Detection**: Gracefully handles browsers without Web Speech API support\n- **Responsive Design**: Adapts to mobile and desktop layouts\n- **Accessibility Compliance**:\n  - ARIA labels and descriptions\n  - Keyboard navigation\n  - Focus indicators\n  - Live region announcements\n\n## Component Structure\n\n```\nscreen-reader/\n├── screen-reader.component.ts          # Component logic\n├── screen-reader.component.html        # Template\n├── screen-reader.component.scss        # Styling\n├── screen-reader.module.ts             # Module declaration\n├── SCREEN_READER_INTEGRATION.md        # Integration guide\n└── README.md                           # This file\n```\n\n## How It Works\n\n### 1. Component Initialization\n- Checks browser support for Web Speech API\n- Loads user's accessibility settings (speech rate, language)\n- Initializes the SpeechSynthesis instance\n\n### 2. Content Extraction\n- Extracts main page content from common content containers:\n  - `<main>` element\n  - `#main-content` ID\n  - `[role=\"main\"]` attribute\n  - Falls back to `document.body`\n- Removes:\n  - Script and style tags\n  - Navigation elements\n  - Hidden elements (`aria-hidden=\"true\"`)\n- Limits content to 5000 characters to prevent extremely long readings\n\n### 3. Text-to-Speech\n- Uses Web Speech API's SpeechSynthesisUtterance\n- Configurable speech rate and language\n- Event handlers for start, end, and error states\n- Pause and resume functionality\n\n## Usage\n\n### Basic Integration\n\n1. **Import the Module**:\n```typescript\nimport { ScreenReaderModule } from './accessibility/screen-reader/screen-reader.module';\n\n@NgModule({\n  imports: [ScreenReaderModule]\n})\nexport class AppModule { }\n```\n\n2. **Add to Template**:\n```html\n<ds-screen-reader></ds-screen-reader>\n```\n\n3. **User Interaction**:\n   - Click the icon to toggle screen reader\n   - Use control buttons to manage playback\n   - The component automatically reads main page content\n\n### Configuration\n\nUsers can configure screen reader settings through the accessibility settings:\n\n```typescript\n// Get screen reader settings\nthis.accessibilitySettingsService.get('screenReaderEnabled').subscribe(enabled => {\n  console.log('Screen reader enabled:', enabled);\n});\n\nthis.accessibilitySettingsService.getAsNumber('screenReaderRate', 1).subscribe(rate => {\n  console.log('Speech rate:', rate);\n});\n\nthis.accessibilitySettingsService.get('screenReaderLanguage', 'en-US').subscribe(lang => {\n  console.log('Speech language:', lang);\n});\n```\n\n## API Reference\n\n### Component Methods\n\n- **toggleScreenReader()**: Toggle screen reader on/off\n- **startReading()**: Start reading page content\n- **stopReading()**: Stop the current reading\n- **pauseReading()**: Pause the current reading\n- **resumeReading()**: Resume paused reading\n\n### Component Properties\n\n- **isActive**: Boolean indicating if screen reader is currently speaking\n- **isSupported**: Boolean indicating browser Web Speech API support\n\n## Styling\n\nThe component uses:\n- **Position**: Fixed, left side, 50% from top\n- **Size**: 50px diameter main button\n- **Colors**: UCU brand gradient (blue to red)\n- **Effects**:\n  - Hover scale animation\n  - Active pulse animation\n  - Smooth transitions\n  - Focus indicators for accessibility\n\n### Responsive Adjustments\n\n- **Mobile** (< 768px):\n  - Positioned at bottom-left\n  - Slightly smaller button (45px)\n  - Control buttons displayed horizontally\n\n## Browser Support\n\n| Browser | Support | Version |\n|---------|---------|----------|\n| Chrome | ✅ | 25+ |\n| Firefox | ✅ | 49+ |\n| Safari | ✅ | 14.1+ |\n| Edge | ✅ | 79+ |\n| Opera | ✅ | 27+ |\n| IE 11 | ❌ | Not supported |\n\n## Accessibility\n\n- **Keyboard Navigation**:\n  - Tab to focus buttons\n  - Enter/Space to activate\n  - Escape to close\n\n- **ARIA Labels**:\n  - All buttons have descriptive labels\n  - Live region for control status\n  - Role attributes for semantic HTML\n\n- **Focus Management**:\n  - Clear focus indicators\n  - Focus trap within controls when active\n  - Return focus after closing\n\n## Performance Considerations\n\n1. **Content Extraction**: Clones DOM and removes unnecessary elements\n2. **Character Limit**: Caps content at 5000 characters\n3. **Memory**: Properly cleans up on component destroy\n4. **Event Handling**: Uses RxJS takeUntil for proper subscription cleanup\n\n## Future Enhancements\n\n- [ ] Add support for custom content selection\n- [ ] Implement voice selection UI\n- [ ] Add reading history\n- [ ] Support for reading specific sections\n- [ ] Integration with PDF reader\n- [ ] Customizable keyboard shortcuts\n- [ ] Reading speed presets\n- [ ] Language auto-detection\n\n## Troubleshooting\n\n### Screen Reader Not Showing\n- Check browser support (see Browser Support section)\n- Verify component is added to template\n- Check browser console for errors\n\n### Audio Not Playing\n- Check browser's speaker volume\n- Verify Web Speech API is available\n- Check accessibility settings for language/rate configuration\n- Try different browser or update to latest version\n\n### Content Not Reading\n- Verify page has main content area\n- Check if content is hidden (`aria-hidden=\"true\"`)\n- Verify text content exists (not just images)\n- Check browser console for errors\n\n## Testing\n\nTo test the screen reader component:\n\n```typescript\n// Test component initialization\nit('should initialize with browser support', () => {\n  const component = new ScreenReaderComponent();\n  expect(component.isSupported).toBe(true);\n});\n\n// Test content extraction\nit('should extract page content', () => {\n  const content = component['extractPageContent']();\n  expect(content).toBeTruthy();\n});\n\n// Test screen reader toggle\nit('should toggle screen reader state', () => {\n  component.toggleScreenReader();\n  expect(component.isActive).toBe(true);\n});\n```\n\n## License\n\nPart of the UCU Institutional Repository\n
